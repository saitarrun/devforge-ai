# /sdlc-retrospective

Blameless retrospectives that drive action items and continuous improvement.

## Usage

```
/sdlc-retrospective "auth-outage-2024-01-15"
/sdlc-retrospective "release-v2.1.0"
```

## Process

1. **Engineering Manager + Tech Lead** → Facilitate
2. **Team** → Share observations (blameless)
3. **Group** → Identify root causes (systems, not people)
4. **Create** → Action items (owner, deadline)
5. **Track** → Follow-up in 2 weeks

## Output

- Blameless summary of what happened
- Root cause analysis (ask "why?" 5 times)
- QUANTS metrics (quality, attention, toil, time, satisfaction)
- Action items (specific, measurable, deadline)
- Team notes and lessons learned

See **skill-organizational-health** and **skill-incident-management** for details.
